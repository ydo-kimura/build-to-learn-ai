# 要件とユニットのマッピング (Requirements to Units Map)

本ドキュメントは、プロジェクトの要件（AIエンジニアとしての実力証明と学習）が、各ユニット（カリキュラム）にどのようにマッピングされているかを示します。ユーザーストーリーフェーズはスキップされたため、要件定義に基づく高次目標とのマッピングを行います。

## 実装パターンの要件マッピング

| 要件 / 目標 (Requirements) | 担当ユニット (Assigned Units) | 達成方法 (Implementation Approach) |
| :--- | :--- | :--- |
| **REQ-1: 機械学習/LLMの仕組みの理解**<br>ブラックボックスとしての利用からの脱却 | Unit 1〜18, Unit 20 | 古典的ML（scikit-learn）からPyTorch、CNN、RNN、Attentionまで、フルスクラッチ実装やミニPoCを通じ裏側の数理・仕組みを実践的に学ぶ。 |
| **REQ-2: RAGの構築**<br>社内文書検索等の実用的なユースケース | Unit 20, Unit 21 | まず自力でベクトルDBとAPIを結合し（Unit 20）、その後にLangChainを用いた堅牢なシステム（Unit 21）へ昇華させる。 |
| **REQ-3: プロンプトチェーンの構築**<br>複雑な推論を伴う業務自動化 | Unit 22 | LLMの出力をパースし、次のプロンプトの入力に渡すパイプラインを実装し、段階的な推論タスクを解かせる。 |
| **REQ-4: チャットボットの構築**<br>対話履歴とコンテキストの管理 | Unit 23 | トークン数の制限を考慮したメモリ管理手法（スライディングウィンドウや要約メモリ等）を実装したチャットUIを構築する。 |
| **REQ-5: AIエージェントの構築**<br>ツール呼び出しによる自律動作 | Unit 24 | ユーザーの意図を解釈し、適切な外部関数（Tools）を呼び出し、結果を統合して返すReActパターンの基礎を実装する。 |

## リクルーター向け可視化目標 (Visibility Goals)

- **GitHub可視性**: 各Unitは独立したディレクトリとして管理され、それぞれの `README.md` に学習内容とPoCの実行方法が記載されます。
- **技術ブログ**: 特にUnit 5 (XGBoost), Unit 9 (NN from Scratch), Unit 14 (ResNet), Unit 18 (Transformers), Unit 21 (LangChain RAG), Unit 24 (Agent)の完了時には、学習と実装の過程をアウトプットし、リクルーターへのアピール材料とします。
