# ユニット（カリキュラム）生成計画 (Unit of Work Plan)

本プロジェクトにおける「Unit（ユニット）」は、 **学習・実践・アウトプットを行うための各技術トピック（カリキュラムの1章分）** に相当します。この計画では、どのようなトピックをどのような順序で進めるかの分解方針を決定します。

## 実行ステップ (Execution Steps)

- [x] `aidlc-docs/inception/application-design/unit-of-work.md` の生成（各トピックの定義とスコープ）
- [x] `aidlc-docs/inception/application-design/unit-of-work-dependency.md` の生成（トピック間の学習順序・依存関係）
- [x] `aidlc-docs/inception/application-design/unit-of-work-story-map.md` の生成（要件とトピックのマッピング）
- [x] コード構成戦略（ディレクトリ構造など）のドキュメント化
- [x] すべてのトピックの境界と依存関係の検証

## 質問事項 (Clarification Questions)

最適なカリキュラム（トピック群）を定義するため、以下の質問にご回答ください。回答は `[Answer]:` の後に直接ご記入いただけます。

### 1. 学習の順序とアプローチ (Dependencies & Approach)
基礎から積み上げる「ボトムアップ型（PyTorch等の基礎モデルから始め、徐々にLangChain等のLLMへ移行）」と、まずは動くものを作る「トップダウン型（LangChain等のAPI利用から入り、後から仕組みを深掘り）」のどちらのアプローチを好みますか？
[Answer]: ボトムアップ型

### 2. トピックの粒度 (Topic Grouping)
1つのUnit（トピック）の期間や規模感はどの程度を想定していますか？（例：「週末1〜2日で終わる小さなPoCを大量に回す」「1〜2週間かけてじっくり1つのテーマ（例：RAG）を深掘りする」など）
[Answer]: 週末1〜2日で終わる小さなPoCを大量に回す

### 3. 最優先の実装パターン (Technical Focus)
AIエンジニアとしての実力を証明するために、「AIエージェント以外」で真っ先に実装・公開してみたいAIパターンはありますか？（例：RAGを用いた社内文書検索、プロンプトチェーンによる業務自動化、テキスト分類モデルなど）
[Answer]: RAGを用いた社内文書検索、チャットボット、プロンプトチェーンによる業務自動化

### 4. リポジトリの構成方針 (Code Organization)
作成するコードのGitHubリポジトリはどのような構成にしたいですか？（例：「トピックごとにディレクトリを分ける1つの巨大な学習リポジトリ（モノレポ）にする」「テーマごとに完全に別のリポジトリを作成する」など）
[Answer]: トピックごとにディレクトリを分ける1つの巨大な学習リポジトリ（モノレポ）にする
