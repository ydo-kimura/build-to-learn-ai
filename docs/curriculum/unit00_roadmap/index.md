# Unit 0: 全体のロードマップと学習の進め方 (Roadmap & Guidelines)

ようこそ！このリポジトリは、機械学習の基礎から始まり、最新のLLM・AIエージェントの構築までを網羅的に学習するためのカリキュラムです。

本リポジトリは、概念を小さなPoC（概念実証）で確認しながら進めるカリキュラムです。目安は基礎ユニットで1〜3時間、CapstoneやAPIを使うユニットで半日〜数日です。所要時間は環境や経験によって変わるため、各ユニットの「到達目標」と「完了条件」を確認しながら進めてください。


## 💡 なぜLLM全盛期に、機械学習やDLから学ぶ必要があるのか？

現在、ChatGPTに代表されるLLM（大規模言語モデル）や自律エージェントがAIの中心となっています。しかし、実務で活躍する一流の「AIエンジニア・システムアーキテクト」を名乗るためには、LLMだけでなく **基礎的な機械学習（ML）やディープラーニング（DL）の知識が不可欠** です。

その最大の理由は、 **「ML、DL、LLM それぞれの技術特性（データの種類、計算速度、コスト、解釈性）によって、実務で最もフィットする（解決すべき）問題が根本的に異なるから」** です。

---

### 📊 ML・DL・LLM：技術特性比較マトリクス

実務においてAIをシステムに組み込む際、アーキテクトは以下のトレードオフ（一長一短）を考慮して技術を選定します。

| 評価軸                          | 機械学習 (ML)                                                                         | ディープラーニング (DL)                                                         | 大規模言語モデル (LLM)                                                                  |
| :------------------------------ | :------------------------------------------------------------------------------------ | :------------------------------------------------------------------------------ | :-------------------------------------------------------------------------------------- |
| **得意なデータ形式**            | **構造化データ** （数値、カテゴリ、顧客DBの売上履歴など）                             | **非構造化データ** （画像、音声、波形、動画など）                               | **自然言語（テキスト）** 、プログラムコード、複雑な対話                                 |
| **推論速度**                    | 高速になりやすい（数ミリ秒〜数十ミリ秒の例）                                          | モデルとハードウェアに依存（数十ミリ秒〜数百ミリ秒の例）                        | API待ちやモデルに依存し、比較的遅くなる場合がある                                       |
| **計算コスト（インフラ）**      | CPUで動作しやすく比較的安価                                                           | GPUやモデルサイズに依存                                                         | API従量課金または専用GPUなど、構成によって大きく変わる                                  |
| **説明可能性 (Explainability)** | **比較的高い** （係数や特徴量重要度を調べやすいが、完全な説明を保証するものではない） | **一般に低め** （多層ニューラルネットの判断根拠を追加手法で分析する必要がある） | **低い** （ハルシネーションや出力の揺らぎがあり、根拠の正しさを別途検証する必要がある） |
| **導入の難易度**                | **中程度** （データの前処理や特徴量設計が必要）                                       | **高め** （データ、計算資源、学習設計が必要）                                   | **API利用の開始は容易だが、本番運用は難しい** （評価、コスト、権限、障害対応が必要）    |

### 🎯 適材適所の技術選定（フィットマップ）

実務におけるシステムアーキテクトとして、各技術をどのように適材適所で使い分けるべきか、その極意を整理します。

#### 1. 機械学習 (ML) がフィットする領域（第1章）

- **代表的なユースケース（活用例）** : 顧客の退会予測、売上・在庫需要の予測、不動産の価格査定、時系列での価格最適化。
- **選定理由** : 企業が保有する表形式データでは、ML（特に木系モデル）が強いベースラインになることが多く、推論速度や特徴量分析の面でも扱いやすい傾向があります。ただし、最適な手法はデータ量、評価指標、制約、運用環境で変わります。
- **⚠️ LLMで解こうとするアンチパターン** :
  「顧客の過去の購入履歴数値」をLLMにテキストで渡して予測させる方法は、専用の教師ありモデルと比較して、再現性・コスト・評価のしやすさで不利になる場合があります。まず専用モデルをベースラインにし、LLMを使う明確な理由があるか検証しましょう。

#### 2. ディープラーニング (DL) がフィットする領域（第2章・第3章前半）

- **代表的なユースケース（活用例）** : 工場の生産ラインにおける製品の傷検知（画像認識）、医療画像の癌検知、音声コマンドの認識、機械の異常波形検知。
- **選定理由** : 画像や波形では、手作業の特徴量設計よりも、DL（CNN等）に特徴抽出を学習させる方が有効な場合があります。ただし、データ量・遅延・精度・保守性を比較して選択します。
- **⚠️ LLMで解こうとするアンチパターン** :
  高速な生産ラインで毎秒多数の画像を処理する場合、クラウドLLM APIは遅延・コスト・接続障害が制約になる可能性があります。その場合は、エッジ上の軽量モデルを含む複数案を測定して選びます。

#### 3. 大規模言語モデル (LLM & Agent) がフィットする領域（第4章・第5章）

- **代表的なユースケース（活用例）** : 契約書からの特定項目の抽出・JSON構造化、カスタマーサポートの対話自動化、仕様書からのソースコード自動生成、自律的なツール実行・自動化。
- **選定理由** : 自然言語の変換・要約・対話や、ツールを組み合わせる柔軟な処理ではLLMやエージェントが有力な候補になります。一方、重要な業務判断は、構造化データ、決定的なルール、承認フロー、評価器と組み合わせて使います。

---

### 💡 結論：なぜ私たちは機械学習（ML）から学び始めるのか？

最新のAIシステムは、LLMだけで構築されることはありません。実務では、 **「MLやDLの軽量な予測モデルが裏側で高速にデータを処理し、その結果をLLMが受け取って人間と対話する」** といった **ハイブリッド設計** が基本になります。

1. **適材適所の技術選定** : 各技術の「特性」と「限界」を正しく見極め、無駄なインフラコストや応答速度の低下（デグレード）を防ぐアーキテクト視点を養います。
2. **モデル内部の理解** : LLMの心臓部（AttentionやTransformer）は、ディープラーニング（DL）や自然言語処理（NLP）の進化の歴史の上に立っています。第1〜3章を学ぶことで、最新のLLMやエージェントの内部挙動を論理的に掌握し、実務での高度なチューニングが可能になります。

## 📝 学習の進め方 (How to Learn)

各Unitの `index.md` は、基本的に以下の4ステップ構成です。プロンプト設計など、概念を分けた方が理解しやすいUnitでは、実装前に補助セクションを挿入しています。ご自身の環境（Jupyter Notebook等）を開き、コードを写経・実行しながら進めてください。

### 各Unitの開始時に確認すること

各Unitには、次の4点を最初に自分で書き出してください。

1. **到達目標** : 用語を説明できること、コードを動かせること、評価結果を解釈できること
2. **前提知識** : 直前のUnitで必要な概念、Python、数学、API設定など
3. **完了条件** : 実装例が動くこと、実践課題を一部でも自力で変更できること、結果を説明できること
4. **所要時間** : 基礎は1〜3時間、Capstoneは半日〜数日を目安にすること

### 1. [トピック名] (概念の理解)

まずはそのUnitで学ぶアルゴリズムや技術の背景、数理的な基礎、メリット・デメリットを理解します。初心者にもわかりやすいように図解や具体例を交えて解説されています。

### 2. 実装例 (Implementation Example)

メインとなるデータセットを用いた「ベースとなる実装コード」とその詳しい解説です。手元にコードをコピーして実行し、パラメータを変えながら挙動の違いを実験してみてください。

### 3. 実践 (Practice)

「2. 実装例」で学んだ知識を使って、別のデータセットに取り組むフェーズです。初学者は、まず写経、次に一部変更、最後に自由実装の順で進めてください。自力で完遂できない場合も、ヒントや答え合わせを使って、なぜその実装になるかを説明できれば学習上の達成です。

### 4. 答え合わせ (Answer Key)

実践（課題）の解答コードと解説です。`<details>`タグで隠されているため、自分の実装が終わるまでクリックせずに挑戦してみてください。解答と自分のコードを比較し、より良い書き方がないかを確認します。
